## Stack

- **Runtime:** Ruby 3.3
- **Package manager:** bundler
- **Services:** PostgreSQL 18, Redis 8
- **Background worker:** `bundle exec sidekiq`

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) with Compose v2
- GNU Make
- _(optional)_ [asdf](https://asdf-vm.com) for bare-metal runtimes — see `.tool-versions`

## Quickstart

```bash
cp .env.example .env     # then adjust any secrets
make setup               # install toolchain + deps, build images
make dev                 # boot the full stack with hot reload
```

App will be available at http://localhost:3000

## Make targets

| Command | Description |
|---|---|
| `make doctor` | Preflight — check Docker is ready and host ports are free |
| `make setup` | Run doctor, then install toolchain + dependencies, build images |
| `make dev` | Start the full stack with hot reload |
| `make db-seed` | Re-run seed SQL against the database |
| `make test` | Run the test suite in the app container |
| `make lint` | Run linter / formatter checks |
| `make logs` | Tail logs for all services |
| `make down` | Stop & remove containers (data volumes kept) |

## Environment variables

Copy `.env.example` to `.env`. Connection strings use the in-network hostnames from `docker-compose.yml` (e.g. `postgres`, `redis`). **Never commit `.env`** — it is git-ignored.

## Troubleshooting

- **Port already in use:** something else holds `3000`. Stop it or change the host port in `docker-compose.yml`.
- **App starts before the DB is ready:** dependencies are gated on healthchecks via `depends_on`. If you disable that, add a wait-for to your entrypoint.
- **Start completely fresh:** `make down` then `docker compose down -v` to drop data volumes.
